패킷 스위칭의 기본 동작원리인

	-> Store and Forward
	-> 패킷스위칭은 기본적으로 모든 라우터에서 store-and-forawad 방식으로 동작함

	-> 패킷들이 호스트에서 만들어져 라우터에 도달하게되면
	-> 제일 좋은 경로를 따라서 패킷을 보내줘야함
	-> 이를 위해 라우터에서 패킷을 한번 저장하고, 패킷의 정보를 확인함
	-> 이 정보에 따라 가장 좋은 경로로 패킷을 보내게 됨


	각 패킷은 L-bit로 고정
	-> 1. 패킷이 링크를 따라 라우터로 전송됨(라우터에 패킷이 채워짐)
		-> (패킷이 빠져나가는데 걸리는 시간 : transmission delay : L/R ) ; 이동하는데 걸리는 시간은 빛의속도
	-> 2. 라우터에 복원된 패킷의 정보를 확인함
	-> 3. 제일 좋은 경로로 패킷을 목적지로 전송함

p. 23

두가지 딜레이가 있다.

	Transmission delay : 패킷이 빠져나가는데 걸리는 시간
	Propagation delay : 링크를 타고 움직이는 시간 (빛의속도 ; 거의 신경쓰지 않음)

	또 Queuing delay가 있다!!
	-> ex.) 호스트 A,B가 만든 패킷이 라우터를 통해 전송된다고 가정
		-> A,B의 전송속도가 100Mb/s인데 다음 라우터가 전송하는 속도는 훨씬 느리다!
		-> 다음 라우터에서 병목현상이 생김!
		-> 패킷이 링크를 통해 나가기 위해 대기를 서는 queueing이 발생!!

	-> arrival rate가 transmission rate보다 빠르면 발생하는게 queueing loss이다.
		-> 이러면 버퍼가 꽉차서 패킷이 사라질 수 있다!!!!

서킷 스위칭 방식

	-> 패킷단위로 보내는 패킷 스위칭과는 다르다
	-> 사전에 자원, 경로등을 미리 예약해두고 목적지까지 데이터를 전달하는 방식
		-> 미리 물리적으로 회선이 설치되어 있음 (end-to-end 방식)
		-> 연결된 두 단말은 할당된 회선만을 사용한다.
	-> 전송속도(성능)이 보장되는 방식이다.
	-> 사용하지 않는 회선은 다른 단말이 사용할 수 없으므로 낭비되는 단점이 있다.
	-> 주로 전화에서 이런 방식을 사용한다

	=> 인터넷 : 패킷스위칭 / 전화 : 서킷스위칭
