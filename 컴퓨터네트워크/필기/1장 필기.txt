인터넷이란 무엇인가?

	-> 너트와 볼트를 생각해 보자
	-> 호스트 : 네트워크 앱을 수행시킬 수 있는 단말기
	-> 네트워크는 데이터를 전달하는데 사용
	-> 단말기는 사용자와 상호작용하는데 사용

	유선 링크와 무선 링크가 있다.
	-> 전송율? 전송속도? : bandwidth와 혼용되어 사용한다.
		-> 유무선에 관계없이 통신매체가 단위시간당 나를 수 있는 신호의 양
		-> 주파수의 대역폭이 얼마나 되는가가 Bandwidth이다.
		-> bandwidth가 데이터의 전송속도를 결정한다.

	라우터?
	-> 데이터가 경로를 통해 들어오면 어디로 보낼지(forwarding) 정하는 역할을 한다
	-> packet : 데이터를 덩어리로 뭉친것

인터넷의 구성요소

	-> 하늘색덩어리 : 네트워크 / 하늘색을 다 모으면 인터넷

	-> 프로토콜 : 소프트웨어의 형태로 구현
		-> 신호를 주고받을때 메시지의 형태로 주고받음
		-> 이 메시지를 어떤 방식,절차를 거쳐서 교환하는지에 대한 규약
		-> ex. TCP, IP, HTTP, Skype, 802.11
	-> 인터넷 표준
		-> RFC : 인터넷을 업데이트 하는과정에서의 요청서
		-> IETF : 규격화 작업을 하는 기구 (비영리단체) / 이 단체의 전무가들이 모여 RFC를 만듬

프로토콜이란???

	-> 기계는 프로그래밍된 대로만 움직인다.
	-> 어떤 절차에 따라서 통신을 할지 명확하게 지정해 주어야함
	-> 모든 인터넷 통신은 프로토콜에 따라야 한다.

	-> 메시지를 어떻게 보내고 어떻게 받는지에 대한 정확한 규약이다.
	-> 메시지의 형태를 정의하고, 메시지가 어떤 순서에따라 주고받아지는지
		, 메시지를 받았을때 어떤 action을 취해야하는지 정해 놓은 규격.

Residential network access의 한 방식

	-> Digital Subscriber Line (DSL)	

	-> DSL : 전화선을 통한 인터넷 연결
	-> 모뎀을 통해 디지털을 아날로그신호로 바꿈
	-> 전화신호와 구분하기 위해 splitter가 존재
	-> DSLAM이 이를 받아 전화신호와 분리한다.

	-> 각 집마다 전화선이 존재함
	-> 보낼때는 초당 2.5메가비트, 받을때는 초당 24메가비트

데이터의 패킷 전송

	-> 데이터는 보통 패킷단위로 묶여 전송된다.
	-> R : 전송 속도
	-> L : 패킷의 길이

	-> 전송 딜레이 = (패킷의 길이)/(전송속도)
		-> sec = L/R
	-> 비트가 네크워크 선까지 진입하는데 걸린시간이지, 선을따라 패킷까지 가는 시간이 아님!!!
